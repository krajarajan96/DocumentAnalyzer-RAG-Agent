# 📄 Dynamic Document Analyzer RAG Agent

An intelligent, web-based Retrieval-Augmented Generation (RAG) assistant built using **LangGraph**, **Streamlit**, and **ChromaDB**. This application allows users to dynamically upload any PDF document, builds an in-memory vector database on the fly, and utilizes a state-machine agent powered by Groq's Llama-3.1 model to answer complex questions with precise document context citations.

---

## 🚀 Getting Started

Follow these step-by-step instructions to isolate your environment, install the correct package matrix, and stand up the service locally.

### 1. Environment Setup

To avoid conflicting global packages, create an isolated virtual environment (`.venv`) inside the root directory of this project.

#### Step A: Create the Virtual Environment
If your machine relies on Xcode Python or a network-bound setup that causes traditional virtual environment creation to freeze, run the following fast bootstrap command:
```bash
# Delete any existing broken environments
rm -rf .venv
```

# Create a clean environment bypassing slow network setups
python3 -m venv .venv --without-pip
Step B: Activate the Virtual Environment
Activate your environment based on your operating system:

Mac / Linux:

Bash
source .venv/bin/activate
Windows (Command Prompt):

DOS
.venv\Scripts\activate.bat
Windows (PowerShell):

PowerShell
.venv\Scripts\Activate.ps1
Confirm activation: Your terminal input line will now display (.venv) at the very beginning.

Step C: Inject and Upgrade Pip
Force-load pip directly from Python's local internal system files and bring it up to date:

Bash
python3 -m ensurepip --default-pip
python3 -m pip install --upgrade pip
2. Install Required Packages
With your virtual environment active, run this single pip installation command to download the exact package matrix required by the application:

Bash
pip install streamlit python-dotenv langchain-huggingface langgraph langchain-groq langchain-chroma pypdf sentence-transformers
3. Configure the .env File
The application relies on the python-dotenv package to securely fetch environmental secret tokens without exposing your keys within the codebase.

Create a file named exactly .env in the root folder of this project (ensure it has a leading dot and no file extension like .txt).

Paste the following template into the file, substituting your actual API secret keys:

Code snippet
GROQ_API_KEY="your_actual_groq_api_key_here"

# Optional (Only needed if you explicitly modify code to use OpenAI models)
OPENAI_API_KEY="your_openai_api_key_here"
🔒 Security Notice: Always add .env and .venv/ to your project's .gitignore file to ensure secrets and heavy localized packages are never committed to your public version control.

💻 Running the Application
Once your virtual environment is active and your keys are configured, launch the local development server:

Bash
streamlit run app.py
Accessing the Interface in Your Browser
Upon launching, the Streamlit process takes ownership of a local port (typically port 8501). It will automatically open your default browser to the correct page. If it does not open automatically, copy and paste one of the URLs printed in your terminal:

On your browser:
  You can now view your Streamlit app in your browser.

  Local URL:   http://localhost:8501
  Network URL: [http://192.168.1.5:8501](http://192.168.1.5:8501)
Note: If port 8501 is already in use by another application, Streamlit automatically flags it and increments to the next open port (e.g., 8502).

🛠 How to Use the App
Upload a Document: Navigate to the side panel on the left labeled Document Control Center. Click Choose a PDF file or drag-and-drop any PDF (e.g., a research paper or financial statement).

Wait for Embedding Initialization: The app will display a spinner indicating that the HuggingFaceEmbeddings model is processing the text and loading chunks into an in-memory ChromaDB matrix.

Ask Questions: Once the success banner appears, type your question into the chat text input box at the bottom of the main interface.

Observe the Tool Call State: When the agent needs information, an expander box will pop up detailing what query the retriever tool is running.

Review Answers: The final clean answer generated by the model will print in the chat stream, complete with contextual facts pulled directly from your document segments.

Hot-Swap Documents: To inspect a different document, simply drop a new PDF into the sidebar. The system automatically wipes the active RAM memory cluster, clears out the previous chat history, and spins up a brand new vector space instantly.
